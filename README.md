## Research Overview

### Research Question
> *How do different prompting techniques affect summarization quality, factuality, and coherence in AI-generated summaries of academic documents?*

### Objective
The core objective of this study is to systematically analyze the performance of various prompting techniques when summarizing dense academic texts. Each generated summary is evaluated across three core dimensions:
*   **Factuality:** Ensuring the summary remains faithful to the source material without introducing hallucinations.
*   **Coherence:** Assessing the logical flow, readability, and structural integrity of the text.
*   **Overall Quality:** Measuring how well the summary captures essential academic concepts against baseline objectives.

Ultimately, this project aims to identify an optimal, general-purpose prompting strategy that consistently yields high-quality academic summaries.

## Project Structure

The repository is organized into the following components:

*   `prompting_techniques/`: A directory containing `.txt` files, where each file defines a specific prompting strategy under evaluation.
*   `textbooks/`: The local dataset composed of `.json` files. Each file contains the parsed text content of an OpenStax textbook chapter, along with its specific `key_terms` acting as the baseline target objectives.
*   `results/`: A directory containing `.csv` files that store the raw performance data and evaluated metrics generated by the main experiment.
*   `URLs.txt`: A text file compiling all source URLs of the OpenStax textbook chapters utilized in this study.
*   `generate_data.py`: A utility script that reads `URLs.txt`, fetches the contents, and populates the `textbooks/` directory with structured JSON files.
*   `coherence_prompt.txt`: A template file containing the criteria and grading rubric used by the LLM judge to grade text coherence.
*   `generate_summary.py`: A script providing the core function to request and retrieve AI-generated summaries given a specific text and a prompting technique.
*   `coherence_LLMjudge.py`: Contains a pipeline that sends the generated summary along with `coherence_prompt.txt` to the Gemini API. It evaluates the summary on a 1–5 scale, which is subsequently normalized to a 0.0–1.0 range.
*   `factuality.py`: Implements the evaluation pipeline using the **SummaC** library to compute a factuality score between 0.0 and 1.0 against the source text.
*   `quality.py`: Computes the **BERTScore** precision, recall, and F1-measure by comparing the generated summary against the chapter's baseline `key_terms`.
*   `experiment.py`: The main execution engine. It orchestrates the entire pipeline, running all prompting techniques across every textbook chapter, evaluating the responses via the metric modules, and exporting the data.

---

## Methodology & Evaluation Metrics

To rigorously assess the summaries, this framework incorporates state-of-the-art evaluation dimensions:

1.  **Factuality (SummaC):** Utilizing the `summac` framework to detect inconsistencies and enforce strict factual alignment with the original text.
2.  **Coherence (LLM-as-a-Judge):** Leveraging Google's **Gemini API** as an automated evaluator to score structural and linguistic coherence based on a structured rubric.
3.  **Overall Quality (BERTScore):** Leveraging contextual embeddings via **BERTScore** to compute an F1-score, verifying semantic overlap between the summary and the official chapter keywords.

## Getting Started & Usage

### Prerequisites
Ensure you have Python installed along with the required dependencies (such as `bert-score`, `summac`, and the official `google-genai` SDK). 

### Setup API Keys
Before executing the pipeline, you must supply your API credentials. Open the relevant scripts (`coherence_LLMjudge.py`, `generate_summary.py`) and replace the credential placeholders with your valid API keys:
```python
# Replace placeholder with your real Google Gemini API Key
API_KEY = "YOUR_ACTUAL_GEMINI_API_KEY"
```

### Execution
To download the text corpus and compile the target JSON files, run:
```bash
python generate_data.py
```

To execute the complete benchmarking experiment across all prompting techniques and generate the results, run:
```bash
python experiment.py
```
Upon completion, the final performance tables will be saved inside the `results/` folder as `.csv` files.
